# Maintainer Guide

## Responsibilities

- protect the legal model;
- review code for security and quality;
- keep release notes and notices current;
- maintain CI, CodeQL, and branch protection.

## Merge policy

- prefer pull-request-only changes to `main`;
- require reviews and passing checks;
- require resolved conversations;
- avoid direct pushes except for narrowly controlled administrative actions.

## Legal policy

- community code is `AGPL-3.0-only`;
- commercial rights are granted only by separate agreement;
- contributions are covered by the CLA.
