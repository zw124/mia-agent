# Security Policy

## Supported reporting path

Please do not open public issues for unpatched security vulnerabilities.

Report security issues privately to the maintainer through the repository security advisory feature or the designated private contact channel.

## What to include

- affected component and version;
- reproduction steps;
- expected impact;
- proof-of-concept if available;
- whether the issue is already known to be exploited.

## Response goals

Targets, not guarantees:

- initial acknowledgement within 5 business days;
- triage and severity assessment after reproduction;
- coordinated disclosure after a fix or mitigation is available.

## Scope

Examples of in-scope issues:

- auth bypass;
- approval bypass;
- remote code execution;
- secret leakage;
- tenant isolation failures;
- unsafe tool invocation or privilege escalation.

## Out of scope

- theoretical issues without a reproducible path;
- dependency advisories with no practical impact on this codebase;
- social engineering, spam, or policy complaints unrelated to a security flaw.
